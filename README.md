# f2f_bank_tests
Автоматизированные тесты для веб-приложения F2F Bank, имитирующее сервис переводов между пользователями в стиле СБП.

## Структура проекта:
25 автотестов (5 файлов) + `Page Objects` (5 файлов)  
Работа включает в себя файл `NOTES.md` в котором описаны найденные баги при тестировании приложения (4 бага: 2 High, 1 Medium, 1 Low).  
Есть файл `README.md` в котором находится документация проекта с инструкциями по запуску + описание тестов с их приоритетом.  
`playwright.config.ts` - конфигурация тестов


## Настройка:
фрейворк - Playwright  
язык - TypeScript   
браузеры - Chromium, Firefox, WebKit  

## Как запустить приложение:
1. Скачайте приложение из репозитория https://github.com/IrinaStyazhkina/f2f_bank_tests
2. Установите зависимости: `npm install`
3. Запустите приложение: `npm run dev`
4. Откройте в браузере:  http://localhost

##  Как запустить тесты:
### для запуска всех тестов:
```bash
npx playwright test
```

### Только конкретный файл
```bash
npx playwright test login
npx playwright test registration
npx playwright test profile
npx playwright test home
npx playwright test transactions
```

### Только в одном браузере
```bash
npx playwright test --project=chromium
npx playwright test --project=firefox
npx playwright test --project=webkit
```

### Запуск в UI-режиме (визуальный, удобен для отладки):
```bash
npx playwright test --ui
```

### Генерация отчета после прогона:
```bash
npx playwright show-report
```

## краткое описание написанных сценариев и их приоритеты

### registration.spec.ts
- **Critical** - успешная регистрация с валидными данными
- **High** - ошибка при пустой форме
- **Medium** - ошибка при невалидном емейл

**Что проверяет:**
- Пользователь может зарегистрироваться с корректными данными
- Система не позволяет зарегистрироваться с пустыми полями
- Система проверяет формат емейл

**используется:** `RegisterPage.ts`

---

### login.spec.ts
- **Critical** - успешный вход с валидными данными
- **High** - ошибка при неверном пароле
- **High** - ошибка для незарегистрированного емейл
- **High** - ошибка при пустой форме
- **Medium** - ошибка при невалидном емейл
- **Low** - ссылка на регистрацию

**Что проверяет:**
- Зарегистрированный пользователь может войти
- Система проверяет пароль
- Система проверяет существование пользователя
- На странице есть ссылка на регистрацию

**используется:** `LoginPage.ts и RegisterPage.ts`

---

### home.spec.ts
- **Critical** - успешный перевод с валидными данными
- **High** - ошибка при невалидном номере телефона
- **High** - ошибка при отрицательной сумме
- **Medium** - ошибка при пустой сумме
- **Medium** - ошибка при сумме больше баланса
- **Low** - отмена перевода

**Что проверяет:**
- Пользователь может отправить перевод
- Система проверяет номер телефона
- Система не позволяет отправить отрицательную сумму
- Система проверяет наличие средств
- Можно отменить перевод

**используется:** `HomePage.ts и LoginPage.ts и RegisterPage.ts`

---

### transactions.spec.ts
- **Critical** - отображение страницы транзакций
- **High** - показ баланса пользователя
- **High** - кнопка пополнения баланса
- **Low** - таблица транзакций
- **Low** - сообщение когда нет транзакций

**Что проверяет:**
- Страница транзакций открывается
- Пользователь видит свой баланс
- Есть возможность пополнить баланс
- Отображается история операций

**используется:** `TransactionsPage.ts и LoginPage.ts и RegisterPage.ts`

---

### profile.spec.ts
- **Critical** - отображение страницы профиля
- **High** - показ имени пользователя
- **High** - показ email пользователя
- **Medium** - показ фамилии пользователя
- **Medium** - показ баланса на странице профиля

**Что проверяет:**
- Страница профиля открывается
- Отображаются данные пользователя (имя, фамилия, email)
- Пользователь видит свой баланс

**используется:** `ProfilePage.ts и LoginPage.ts и RegisterPage.ts`
